# 碎片接收流程

## 觸發條件

當業萱透過 Telegram 丟進碎片（截圖、網址、文字、對話紀錄）時。

## 執行步驟

1. **接收碎片**
   - 記錄來源類型：截圖 / 網址 / 文字 / 對話
   - 記錄日期時間
   - 存入 `inbox/raw/`（不修改、不刪除）

2. **預摘要**
   - 用一段話總結碎片的核心價值
   - 標記關鍵詞（3-5 個）
   - 判斷 Lion Score（變現/可複製/長期/顧問/內容，1-5 分）

3. **自動連結（雙向連結，LLM Wiki 標準）**

   ### 3a. 前置：讀 index.md
   - 讀 `index.md` 掌握現有頁面清單

   ### 3b. 正向連結（outbound）
   - 搜尋 `wiki/` 裡有沒有相關內容（用 `search_files` 搜關鍵詞）
   - 建立雙向連結 `[[相關概念]]`（至少 2 個）
   - 優先順序：
     1. `[[wiki/concepts/相關概念]]`
     2. `[[wiki/entities/相關實體]]`
     3. `[[wiki/cases/案例庫/相關案例]]`
     4. `[[wiki/synthesis/相關綜合]]`
     5. `[[index.md]]`、`[[hot.md]]`
   - 如果沒有相關內容，標記「新主題候選，可能需要建立概念頁」

   ### 3c. 反向連結（inbound）
   - 找出哪些既有頁面提到這個碎片的主题/實體/概念
   - 在那些舊頁面上追加：`- [[新頁面路徑|新主題]] （YYYY-MM-DD 新增/更新）`
   - 若舊頁面沒有關聯區塊，在頁面末尾新增

   ### 3d. Orphan 防範
   - 新頁面必須有 ≥2 個 outbound wikilinks
   - 新頁面必須有 ≥1 個 inbound wikilinks（由其他頁面鏈來）
   - 無法滿足時標記「需補充連結」

   ### 3e. Frontmatter
   - 新頁面必須包含 YAML frontmatter：
   ```yaml
   ---
   title: 頁面標題
   created: YYYY-MM-DD
   updated: YYYY-MM-DD
   type: note | concept | entity | synthesis
   tags: [tag1, tag2]
   sources: [inbox/raw/來源檔名.md]
   ---
   ```
   - 更新既有頁面時只改 `updated`，不改 `created`

4. **存入 processed**
   - 寫入 `inbox/processed/YYYY-MM-DD-來源類型-摘要.md`
   - 格式：
     ```markdown
     # 📥 碎片標題
     - Date: [[{{DATE}}]]
     - Source: 截圖/網址/文字/對話
     - Tags: #標籤1 #標籤2
     - Lion Score: 變現 X / 可複製 X / 長期 X / 顧問 X / 內容 X
     
     ## 預摘要
     一段話總結核心價值
     
     ## 正向連結
     - [[wiki/concepts/相關概念1]]
     - [[wiki/entities/相關實體2]]
     
     ## 反向連結更新
     - [[wiki/concepts/舊頁面1]] — 已追加此碎片連結
     - [[wiki/entities/舊頁面2]] — 已追加此碎片連結
     
     ## 原始內容
     [截圖/網址/文字/對話紀錄]
     
     ## 業萱的判斷
     [等待補充]
     ```

5. **回傳 Telegram 通知**
   ```
   📥 收到碎片
   
   來源：截圖 / 網址 / 文字
   預摘要：一段話總結
   正向連結：[[wiki/相關概念1]], [[wiki/相關概念2]]
   反向連結：已更新 [[wiki/舊頁面1]], [[wiki/舊頁面2]]
   
   已放入待整理區。有空再補充你的看法。
   ```

## AI 行為規則

### 自動存入 wiki（不需要業萱同意）

以下情況自動把碎片從 `inbox/processed/` 移到 `wiki/`：

- **Lion Score ≥ 4（任何維度）**：變現、可複製、長期、顧問、內容任一項 4 分以上
- **商業判斷 / 策略討論**：業萱表達了對某個主題的看法或決定
- **成功/失敗案例**：有學習價值的實戰經驗
- **重複出現 3 次以上的話題**：自動提名 skill candidate

### 需要業萱確認才存

- **Lion Score 2-3**：中等價值，給建議等回覆
- **低價值**：不存，或只存到 `wiki/notes/` 備查

### 存入後的通知格式

```
✅ 已存入知識庫

位置：wiki/[category]/[topic].md
內容：[摘要]
正向連結：[[相關概念1]], [[相關概念2]]
反向連結：已更新 [[舊頁面1]], [[舊頁面2]]

（回覆「撤銷」可移除）
```

## 知識庫結構規則

- `inbox/raw/`：原始碎片，AI 只讀不改
- `inbox/processed/`：AI 預摘要 + 業萱的回答
- `wiki/`：編譯後的知識庫，AI 維護，業萱只讀不編輯
- 新碎片進來 → **更新現有頁面**，不是新增
- 每個 wiki 頁面必須包含「業萱的判斷邏輯」欄位
- 每個 wiki 頁面必須有 frontmatter（title, created, updated, type, tags, sources）

## 雙向連結速查

| 動作 | 說明 | 最低要求 |
|---|---|---|
| 正向連結 | 新頁面 → 鏈到既有頁面 | ≥2 個 `[[wikilinks]]` |
| 反向連結 | 既有頁面 → 鏈回新頁面 | ≥1 個 inbound |
| Orphan 檢查 | 確保新頁面不是孤島 | outbound ≥2 且 inbound ≥1 |
| Index 同步 | 新頁面 → 加入 index.md | 每頁一條目 |
| Frontmatter | 每頁開頭 YAML | title, created, updated, type, tags, sources |

## 注意事項

- 不要先搜 `raw/`，搜尋順序：`index.md -> hot.md -> wiki/synthesis -> wiki/concepts -> wiki/notes -> inbox/processed`
- `raw/` 永遠不可修改、不可刪除、不可覆蓋
- `outputs/` 不是終點，必須回流到 `wiki/`
- 同一主題重複出現 3 次以上，提名是否升級為 Skill
