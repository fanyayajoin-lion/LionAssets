# 每日蒸餾 Workflow

## 目的

把今天新增的 raw、notes、outputs 消化成可查、可用、可延伸的知識。

## 流程

1. 讀 `index.md`、`hot.md`、`log.md` 最近紀錄。
2. 檢查今日新增或修改的 `raw/`、`wiki/notes/`、`outputs/`。
3. 每則資料判斷：保留、合併、升級 synthesis、提名 skill、封存。
4. 更新相關 `wiki/concepts/`、`wiki/entities/`、`wiki/synthesis/`。
5. 對高價值資料補 Lion Score。
6. 更新 `index.md` 的入口與 `hot.md` 的待整理/待輸出。
7. append `log.md`。

## 輸出格式

- 今日新增洞察
- 可輸出題目
- 可產品化素材
- skill candidate
- 明日優先整理

## 吞吐量追蹤（2026-07-18 新增 — 會議決議）

每日萃取日誌必須追加以下統計欄位：

```
## 📊 碎片處理吞吐量
| 指標 | 數值 |
|------|------|
| raw/ 峰值（今日最高） | X 筆 |
| raw/ 最低點（今日） | X 筆 |
| 平均處理時間 | X 分鐘/筆 |
| 成功入庫 wiki | X 筆 |
| 跳過（重複/Low Score） | X 筆 |
| 總處理量 | X 筆 |
```

這些數據每週萃取日誌會匯總到 `wiki/notes/每週吞吐報告.md`，用於長期趨勢追蹤。
