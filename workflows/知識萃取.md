# 知識萃取流程

## 觸發條件
業萱回覆碎片判斷邏輯，或有新的決策/策略討論。

## 執行步驟

1. **讀取碎片** — 從 `inbox/processed/` 讀取，檢查「業萱的判斷」欄位
2. **萃取判斷邏輯** — 提取業萱的判斷、故事、想法（最高優先級）
3. **決定存放位置**
   - 新概念 → `wiki/concepts/`
   - 案例/經驗 → `wiki/notes/`
   - 跨來源洞察 → `wiki/synthesis/`
   - 流程/方法 → `workflows/`
   - 可重複能力 → `skills/`（提名 skill candidate）
4. **更新現有頁面** — 有相關就更新，不新建
5. **通知業萱** — 告知存入位置和內容摘要

## 知識複利原則
- 新碎片進來 → 更新現有頁面（不是新增）
- 業萱的判斷邏輯是核心價值
- 知識庫會變成「業萱的思維模型庫」

## 注意事項
- 不要只摘要資料；要補上商業判斷、可複用框架與下一步
- 不要把 `outputs/` 當終點，輸出必須回流
- 同一類工作出現 3 次以上，提名 skill candidate
