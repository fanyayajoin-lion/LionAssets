# 知識入庫 Workflow

## 使用時機

AI 閱讀員已在 `inbox/processed/` 產生老闆理解版，需要判斷是否進正式 wiki 時使用。

## 知識提升邏輯

```text
inbox/raw -> inbox/processed -> wiki/notes -> wiki/concepts 或 wiki/synthesis -> skills 或 outputs
```

## 判斷規則

- 單篇有價值：進 `wiki/notes/`。
- 多篇都在講同一概念：整理進 `wiki/concepts/`。
- 多來源形成新判斷：整理進 `wiki/synthesis/`。
- 同類工作出現 3 次以上：提名 `skills/`。
- 能對外發文或簡報：進 `outputs/`，並回流 wiki。

## 入庫 metadata

每篇 note 至少包含：

```yaml
---
type:
source:
topics:
entities:
status:
value_level:
created_at:
---
```

## 入庫後必做

- 更新或確認相關 `wiki/concepts/`。
- 更新或確認相關 `wiki/entities/`。
- 必要時更新 `wiki/synthesis/`。
- append `log.md`。
- 若是近期重點，更新 `hot.md`。
- 若是重要入口，更新 `index.md`。
- Discord `#knowledge-updates` 顯示新增項目。
