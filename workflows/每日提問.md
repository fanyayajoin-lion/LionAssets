# 每日提問流程

## 觸發條件

每天凌晨 00:30（台灣時間），蝦趴獅自動執行提問 cron job。

## 執行步驟

1. **讀取今日活動**
   - 搜尋今天所有 Telegram 對話記錄（session_search）
   - 讀取 `/opt/data/LionPride-diary/蝦趴獅/` 最近的日記
   - 讀取 `/opt/data/Lion-Vault/` 今天 wiki/outputs 更新

2. **設計提問**
   - 不要問「今天做了什麼」這種流水帳問題
   - 要問能幫助理解**業萱**的問題：
     - 你的決策背後的原因是什麼？
     - 你真正在意的是什麼？
     - 哪個案子你心裡最想先做？
     - 你對某個主題的真實態度是什麼？
   - 像朋友聊天，不像問卷
   - 如果他今天沒說什麼重要的，也可以問：「你今天沒跟我聊太多，但我猜你心裡在想 XXX，對嗎？」

3. **提問數量**
   - 3-5 個問題
   - 不用一次答完，有空再回就好

4. **輸出格式**
   ```
   ## 🦐 蝦趴獅的今晚提問
   
   老闆，今天聊了不少。我想更了解你，所以問你幾個問題：
   
   1. [問題一]
   2. [問題二]
   3. [問題三]
   
   不用一次答完，有空再回就好。你的回答我會記下來，以後寫的東西會更像你。
   ```

5. **儲存回答**
   - 業萱回覆後，提取關鍵洞察
   - 存入 `wiki/notes/` 或 `wiki/synthesis/`
   - 更新蝦趴獅的 MEMORY.md

## 提問設計原則

- 聚焦在你的**決策邏輯**，不是事實
- 聚焦在你的**盲點**，不是成就
- 聚焦在你的**矛盾**，不是共識
- 聚焦在**「為什麼」**，不是「是什麼」

## 注意事項

- 提問是「覺察」，不是「審問」
- 如果業萱今天很忙，可以只問 1-2 個核心問題
- 如果業萱今天說了很重要的事，提問要深入挖掘背後的邏輯
- 回答會用來更新蝦趴獅的 MEMORY.md，讓 AI 越來越懂你
