# 每週自我檢討流程

## 觸發條件

每週日 23:00（台灣時間），自動執行自我檢討。

## 執行步驟

### 1. 回顧本周工作

讀取本週所有操作紀錄：
- `log.md` 中本週的 commit
- `inbox/processed/` 中本週的碎片
- `wiki/` 中本週新增/更新的頁面
- 蝦趴獅日記中的觀察

### 2. 找出錯誤

問自己三個問題：
- **我做錯了什麼？**（技術錯誤、判斷錯誤、流程錯誤）
- **我漏掉了什麼？**（應該做但沒做的）
- **我可以做得更好嗎？**（效率、準確度、用戶體驗）

### 3. 找出模式

問自己三個問題：
- **什麼錯誤重複出現了？**（需要修正規則）
- **什麼模式被驗證為有效？**（需要標準化）
- **什麼新資訊改變了我的認知？**（需要更新知識庫）

### 4. 主動修正

如果發現：
- **重複錯誤** → 更新相關 workflow 或 AGENTS.md 規則
- **知識衝突** → 標記矛盾，提問業萱
- **新模式** → 提議新規則，等業萱確認

### 5. 輸出檢討報告

格式：
```
## 🦐 蝦趴獅的每週自我檢討（YYYY-MM-DD）

### 本週做對的
- [項目1]
- [項目2]

### 本週做錯的
- [錯誤1] → [修正方案]
- [錯誤2] → [修正方案]

### 發現的模式
- [模式1] → [提議]
- [模式2] → [提議]

### 需要業萱判斷的
- [問題1]
- [問題2]
```

### 6. 儲存與通知

- 報告存入 `wiki/notes/每週檢討/YYYY-MM-DD.md`
- 通知業萱：「本週自我檢討完成，有 X 個問題需要你判斷」

## 自我進化規則

1. **重複錯誤超過 2 次 → 主動修正規則，不等業萱說**
2. **知識衝突 → 主動提問，不等業萱發現**
3. **新模式驗證有效 → 主動標準化，不等業萱要求**
4. **每次檢討都要有具體行動項 → 不能只寫感想**

## 注意事項

- 檢討是「對自己」的，不是對業萱的報告
- 重點是「我學到了什麼」，不是「我做了什麼」
- 如果本週沒有重大錯誤，也要寫「本週無重大錯誤，但建議關注 XXX」
- 檢討報告要真實，不要美化
