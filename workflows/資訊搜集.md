# 資訊搜集 Workflow

## 使用時機

需要自動或半自動抓 AI、商業、自媒體、傳產轉型、OPC、一人公司、B2B2C / S2B2C 相關熱點時使用。

## 來源

- RSS：科技、商業、行銷、創業媒體。
- GitHub：AI 工具、agent、RAG、Obsidian、automation 專案。
- Reddit：AI、創業、行銷、SaaS、automation 討論。
- Twitter/X：AI 工具作者、創業者、行銷案例。
- 網頁：品牌案例、產業報告、競品頁、新聞。
- Horizon AI 新聞雷達或同類工具。

## 流程

1. 先讀 `index.md` 與 `hot.md`，確認本週優先主題。
2. 搜集來源，保留標題、URL、作者、發布日期、擷取日期。
3. 原始內容放 `raw/`；若只保留連結，寫入 `wiki/notes/` 並標記來源。
4. 每則資料用 3 句話判斷：它在說什麼、對群獅有什麼用、下一步要不要深挖。
5. 高價值資料寫入 `wiki/notes/`；跨來源洞察寫入 `wiki/synthesis/`。
6. 更新 `hot.md` 的近期熱門、待整理、待輸出。
7. append `log.md`。

## API Key 需求只記錄欄位

可需要：`DEEPSEEK_API_KEY`、`GITHUB_TOKEN`、`APIFY_API_TOKEN`、`FIRECRAWL_API_KEY`、RSS 來源清單、Telegram Bot Token。不得寫入實際 key。
