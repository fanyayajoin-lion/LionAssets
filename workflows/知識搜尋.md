# 知識搜尋 Workflow

## 使用時機

老闆或 AI 使用 `/find 關鍵字`、查主題、查人物、查觀點、查可輸出內容時使用。

## 搜尋順序

```text
index.md -> hot.md -> wiki/synthesis -> wiki/concepts -> wiki/notes -> inbox/processed
```

不要先搜 `raw/`，因為 `raw/` 是原始資料，不是老闆理解後的知識。只有需要回查原文時才搜尋 `raw/`。

## 常見查詢

```text
/find 個人品牌
/find Justin Welsh
/find 本週值得輸出的觀點
/find 跟顧問式銷售有關的資料
/find needs-review
```

## 回答格式

```markdown
## 找到的重點
- ...

## 相關知識
- [[wiki/synthesis/...]]
- [[wiki/concepts/...]]
- [[wiki/notes/...]]

## 可用觀點
1.
2.
3.

## 下一步建議
- ...
```

## 搜尋後檢查

- 是否找到過期內容？
- 是否有互相矛盾的觀點？
- 是否有孤島 note 需要連到 concept 或 synthesis？
- 是否有可以提名 skill 的重複工作？
