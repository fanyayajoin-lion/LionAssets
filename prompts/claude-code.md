# 好用提示詞庫

## 說明

收集各種工具、AI 助手的實用提示詞。任何人（或任何 AI）找到這個目錄，就知道怎麼更好用這些工具。

---

## Claude Code 提示詞

### 1. 大白話解釋運作流程

**用途：** 讓 Claude Code 用非技術語言解釋它在做什麼、背後的原理是什麼，適合沒有技術背景的人理解代理機制的運作。

**提示詞：**

```
這是我的提示詞：「【在這裡輸入你的任務需求】」

請幫我啟動一個非同步背景代理來執行這項工作，並在它工作的過程中定期查看進度，然後幫我摘要目前發生了什麼事。

我完全沒有技術背景，所以請用最簡單的大白話為我做摘要。請利用「探索代理」（Explore Agent）向我解釋另一個代理正在處理的事情背後的運作原理，好讓我在過程中順便學習。如果背景代理遇到任何錯誤，請立刻停下來告訴我，並引導我修復它。

請記住，我沒有任何技術背景，任何技術術語對我都沒有幫助。
```

**使用方式：** 把「【在這裡輸入你的任務需求】」替換成實際任務描述後貼給 Claude Code。

---

### 2. 需求訪談再動工

**用途：** 在 Claude Code 開始寫程式之前，先透過深度提問釐清需求，避免做出來不是自己要的東西。

**提示詞：**

```
請閱讀這份@SPEC.md，使用AskUserQuestionTool詳細問我關於這裡面的任何事情，包括技術實作、UI & UX、隱憂、權衡等。請確保提出的問題不是那種顯而易見的表面問題。

務必非常深入，持續不斷地問我，直到細節完全齊全為止，再把最終的規格書（Spec）寫入檔案中。
```

**使用方式：** 先準備好 `SPEC.md` 規格文件，再貼這段提示詞給 Claude Code。

---

### 3. 開發計畫做成網頁

**用途：** 讓 Claude Code 把開發計畫用 HTML 呈現，包含原型、資料流圖、重要程式碼片段，方便審查。

**提示詞：**

```
請在一個HTML檔案中建立一份詳盡的實作計畫。請務必在裡面製作一些原型（Mock-ups）、展示數據流（Data Flow），加入我可能需要審查的重要程式碼片段。請確保內容很容易閱讀、消化。
```

**使用時機：** 開發階段初期，讓計畫可視化。

---

### 4. 固化重複任務成技能

**用途：** 把反覆教 Claude Code 的步驟序列轉成可重複使用的技能（Skill），以後直接執行快捷鍵即可。

**提示詞：**

```
請為這個專案建立一個 [/ship] 技能，用來執行 [執行linter語法檢查和測試，然後草擬一份commit訊息]。

第一個中括號是你要用來觸發它的快捷鍵名稱（例如 /ship），第二個中括號填進你每次都懶得重新教Claude的步驟序列。Claude會自動把它寫成技能，你團隊的任何人都可以直接執行這個指令。
```

**使用場景：**
- 每次都要教 linter + test + commit 的流程
- 固定的程式碼審查步驟
- 重複的部署或構建流程

---

### 5. 自動捕捉對話記憶

**用途：** 讓 Claude Code 自動總結本次對話的重點，並建議哪些內容應該寫入 CLAUDE.md / AGENTS.md 作為長期記憶。

**提示詞：**

```
摘要我們這個Session做了什麼，建議有哪些內容應該新增到CLAUDE.md中。
```

**使用時機：** 每次開發或專案工作結束後使用。
