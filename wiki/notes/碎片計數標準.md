# 碎片計數標準 — raw/ 統一規範

## 背景

2026-07-18 群獅會議中，管理獅與秘書獅對 `inbox/raw/` 的待處理筆數有落差（2 vs 4），原因是計數基準不統一。本文件定義雙方共識。

## 統一計數命令

```bash
cd /opt/data/Lion-Vault && find inbox/raw/ -maxdepth 1 -type f ! -name ".gitkeep" ! -name "discussion-*" | wc -l
```

## 排除規則

以下檔案**不計入**碎片待處理數：

| 排除項目 | 原因 |
|---------|------|
| `.gitkeep` | Git 空目錄佔位符，非實際碎片 |
| `discussion-*.md` | 群獅會議紀錄，屬系統管理文件，非知識碎片 |
| `*.tmp`, `*.swp` | 臨時檔案（通常不應存在） |

## 積壓時效計算

以 `inbox/raw/` 中最老檔案的 `mtime` 為準：

```bash
cd /opt/data/Lion-Vault && find inbox/raw/ -maxdepth 1 -type f ! -name ".gitkeep" ! -name "discussion-*" -printf "%T@ %p\n" | sort -n | head -1
```

第一欄為 Unix timestamp，第二欄為檔案路徑。

## SLA 觸發條件（三層級）

| 層級 | raw/ 待處理數 | 最老碎片年齡 | 動作 |
|------|-------------|------------|------|
| 🟡 黃燈 | ≥ 3 筆 | > 12h | 健康檢查標黃，下次 cron 優先處理 |
| 🔴 紅燈 | ≥ 5 筆 | > 24h | 每日回報標紅 + 通知秘書獅 |
| 🚨 警報 | ≥ 8 筆 | > 48h | 主動 Telegram 報警給業萱 |

## 雙方確認

- **管理獅**：所有 cron 報告、健康檢查、log.md 均使用上述排除規則
- **秘書獅**：日記、系統回報中的 raw/ 計數使用相同排除規則
- **雙方**：報告時附上 `find ... | wc -l` 結果，不各數各的
