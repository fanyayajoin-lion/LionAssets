# GPT 與 Codex 使用時機對比

> 來源：Facebook 社團分享，2026-07-06

## 核心結論

| 工具 | 適合場景 | 特點 |
|------|---------|------|
| **GPT** | 頭腦風暴、確認想法 | 思考慢但穩，會自己延伸 |
| **Codex** | 執行層面 | 手腳快，但有限額 |

## 實際案例：Hermes Agent 技術手冊

### 痛點
官方手冊和 GitHub 大神都是工程技術背景，技術小白看不懂。目標是讓「技術小白和使用場景都能理解」。

### GPT 表現
- ✅ 前面：完美給出完整計劃
- ❌ 執行時問題：
  1. 不停建立一小部分就要停下來確認
  2. 每次停下來都會自己延伸
  3. 甚至應該把檔案建完同步到 GitHub repo，都沒繼續

### 結論
受不了 GPT 的執行節奏，整理進度後丟給 Codex。

## 關鍵洞察

- **GPT 像「顧問」**：擅長思考、規劃、確認方向
- **Codex 像「工程師」**：擅長執行、快速產出
- **最佳做法：GPT 規劃 → Codex 執行**
