# AI 工具分層使用法 — GPT vs Codex 的正確分工

> 來源：Facebook 社團分享 + 蒸餾實驗心得
> 處理日期：2026-07-07

## 核心洞察

**AI 工具選型不是比誰強，而是比誰適合當前階段。**

| 階段 | 適合工具 | 原因 |
|------|---------|------|
| 頭腦風暴、確認想法 | GPT（ChatGPT） | 對話流暢、能延伸思考 |
| 執行層面、大量產出 | Codex（或 Hermes Agent） | 手腳快、能直接操作檔案 |
| 判斷邏輯、框架設計 | 強模型（Claude Opus/Sonnet） | 學得多、思考深 |
| 重複執行、落實細節 | 弱模型（Sonnet/Haiku） | 成本低、效率高 |

## 實際案例：Hermes Agent 技術手冊

老闆在做「Hermes Agent 完整研究手冊」時的真實經驗：

1. **GPT 階段**：完美給出完整計劃 ✅
2. **執行卡關**：GPT 不停建立一小部分就停下來確認，每次都自己延伸，甚至想把檔案建完同步到 GitHub 都不繼續
3. **切換 Codex**：整理進度後丟給 Codex，執行速度快很多
4. **痛點**：Codex 有額度限制

## 延伸洞察：AI 蒸餾的本質 = 團隊管理

從蒸餾實驗中提煉出的更高層次框架：

- **強模型 = 主管**：負責判斷力（規劃、框架、邏輯）
- **弱模型 = 下屬**：負責執行（照做、落實）
- 就像帶團隊：主管做判斷，下屬做執行
- 多數人還習慣「一個對話框做完所有事」→ 這是教學機會

## 為什麼強模型判斷力好？

- 學得更多、思考更多、經歷更多
- 量變引起質變
- 不是「同一個對話框做完」，而是「分層使用」

## 顧問可用性

- 可以變成「AI 工具選型診斷框架」
- 問客戶：你現在卡在什麼階段？ brainstorming 還是 execution？
- 幫客戶設計「強模型判斷 + 弱模型執行」的分層工作流
