# Obsidian 全面使用指南與插件推薦

> **標籤**：#Obsidian #筆記軟體 #知識管理 #插件
> **最後更新**：2026-07-14
> **說明**：Obsidian 完整使用方式、核心功能、知識管理方法論與插件推薦清單。

---

## 一、基本認識

| 項目 | 說明 |
|------|------|
| **儲存格式** | Markdown（.md），純文字檔案，未來相容任何筆記軟體 |
| **核心概念** | Vault（倉庫）= 一個資料夾，所有筆記存在裡面 |
| **最大特色** | 雙向連結（wikilink `[[ ]]`），讓筆記之間產生關聯 |
| **平台支援** | Windows / Mac / Linux / iOS / Android，全部免費 |
| **付費選項** | Obsidian Sync（官方同步，$8/月）或自己用 Git/iCloud/Syncthing |

## 二、核心功能（內建，不用裝插件就能用）

| 功能 | 用途 | 開啟方式 |
|------|------|---------|
| **反向連結** | 看哪些筆記提到你現在看的這篇 | 右側面板「反向連結」 |
| **關係圖（Graph View）** | 視覺化所有筆記的連結關係 | 左側面板「關聯圖」 |
| **快速切換** | 快速搜尋並打開筆記 | `Ctrl/Cmd + P` |
| **命令面板** | 執行各種操作（搜尋、插入模板等） | `Ctrl/Cmd + P` |
| **雙欄編輯** | 左右兩邊同時編輯不同筆記 | 右鍵標籤頁 → 「分割」 |
| **備份還原** | 自動備份筆記版本，避免誤刪 | 設定 → 核心外掛 → 檔案復原 |
| **工作區** | 記憶不同的版面配置（如寫作 vs 瀏覽） | 右側面板「工作區」 |

## 三、使用流程（從入門到精通）

### 初級階段：建立基礎習慣

| 步驟 | 做法 | 說明 |
|------|------|------|
| ① 建立 Vault | 選一個本地資料夾作為倉庫 | 建議放在 Git 可追蹤的位置 |
| ② 開始寫每日筆記 | 每天寫一篇，記錄想法與待辦 | 初期最簡單的知識累積方式 |
| ③ 學習基本 Markdown | `#` 標題、`**粗體**`、`[連結](URL)` | 不需要學很多，基本的就夠了 |
| ④ 開始雙向連結 | 用 `[[ ]]` 連結相關筆記 | 不用擔心連錯，後面會整理 |

### 中級階段：優化工作流程

| 步驟 | 做法 | 說明 |
|------|------|------|
| ⑤ 建立模板系統 | 為常見筆記類型建立模板（日誌、會議、書摘） | 配合 Templater 插件更強大 |
| ⑥ 標籤分類 | 用 `#標籤` 做橫向分類，用資料夾做縱向分類 | 標籤比資料夾更靈活 |
| ⑦ 使用 Dataview | 自動匯出符合條件的筆記清單 | 例如：列出所有 tagged #待讀 的筆記 |
| ⑧ 建立知識架構 | 參考 PARA 法則（Project/Area/Resource/Archive） | 不用照搬，找到適合自己的結構 |

### 高級階段：知識複利

| 步驟 | 做法 | 說明 |
|------|------|------|
| ⑨ 漸進式總結 | 對同一主題反覆補充、深化 | 像種樹一樣，越長越大 |
| ⑩ 定期回顧 | 用 Review 插件安排間隔重複複習 | 把筆記變成真正掌握的知識 |
| ⑪ 跨筆記查詢 | 用 Dataview 做複雜查詢與儀表板 | 例如：統計每月的閱讀筆記數量 |
| ⑫ 輸出導向 | 從筆記庫直接產出文章、簡報、報告 | 知識管理的最終目的 |

## 四、知識管理方法論（怎麼組織筆記）

| 方法論 | 核心概念 | 適用場景 |
|--------|---------|---------|
| **PARA** | 按「專案/領域/資源/檔案」分4類 | 有明確工作專案的人 |
| **Zettelkasten（卡片盒）** | 每張卡片一個想法，互相連結 | 研究、寫作、創意發想 |
| **MOC（內容目錄）** | 不靠資料夾，靠「索引筆記」串聯 | 筆記量大的進階用戶 |
| **Evergreen Notes（常青筆記）** | 持續更新同一篇筆記，越寫越深 | 長期學習與知識累積 |

> **與知識庫關聯：** 老闆目前的 Lion-Vault 知識庫自成一派——用 `wiki/concepts/`、`wiki/entities/`、`wiki/synthesis/` 等分類，配合雙向連結和 frontmatter，是一種混合了 PARA 與 Zettelkasten 的實戰風格。

## 五、推薦插件清單

### ⭐ 第一梯隊（強烈建議安裝）

| 插件名稱 | 功能 | 為什麼好用 |
|----------|------|-----------|
| **Templater** | 進階模板引擎 | 自動抓取天氣、日期、股市，創建筆記時自動填充內容 |
| **Dataview** | 資料庫查詢 | 用類似 SQL 的語法查詢筆記，自動生成清單、儀表板 |
| **Periodic Notes** | 週/月/日誌自動生成 | 不用手動建筆記，每天自動生成對應日誌 |
| **Omnisearch** | 全文搜尋 | 搜尋速度更快，支援中文、PDF、圖片文字 |
| **Kanban** | 看板管理 | 像 Trello 一樣管理任務，基於 Markdown |
| **Tasks** | 任務追蹤 | 跨筆記追蹤待辦事項，支援到期日、重複、優先級 |

### 🛠️ 第二梯隊（根據需求選擇）

| 插件名稱 | 功能 | 適合誰 |
|----------|------|--------|
| **Auto Note Mover** | 自動移動筆記到正確資料夾 | 不想手動整理的人 |
| **QuickAdd** | 快速內容捕獲 | 喜歡用快捷方式錄入想法 |
| **Excalidraw** | 手繪風格圖表 | 需要畫流程圖、架構圖 |
| **Notebook Navigator** | 增強版檔案總管 | 想要更好的文件瀏覽體驗 |
| **Local Backup** | 自動備份倉庫 | 怕資料遺失的人 |
| **Review** | 間隔重複複習 | 想把筆記變成長期記憶 |
| **Life Tracker** | 生活數據追蹤 | 用 Obsidian 記體重、運動、習慣 |
| **Journal Bases** | 日誌資料庫視圖 | 想把日誌做成表格檢視 |

### ✨ 第三梯隊（小工具型）

| 插件名稱 | 功能 | 說明 |
|----------|------|------|
| **Doubleshift** | 雙 Shift 打開命令面板 | 開發者友善，不用記快捷鍵 |
| **Linter** | 自動格式化 Markdown | 保持筆記格式整齊一致 |
| **Paste Image Renamer** | 貼上圖片自動命名 | 避免一堆 `image-20260714.png` |
| **Folder Note** | 資料夾附帶摘要筆記 | 每個資料夾頂端放一篇總覽 |
| **Image Context Menus** | 右鍵圖片選單增強 | 快速調整圖片大小、格式 |
| **Simple Embeds** | 美化嵌入內容 | 讓引用筆記看起來更美觀 |
| **Homepage** | 啟動自訂首頁 | 每次開 Obsidian 看到儀表板 |
| **Focus Mode** | 專注模式 | 隱藏 UI，只留編輯區 |

### 🤖 AI 相關插件（2026 趨勢）

| 插件名稱 | 功能 | 說明 |
|----------|------|------|
| **Smart Connections** | AI 自動推薦相關筆記 | 輸入時自動彈出關聯舊筆記 |
| **AI Assistant** | 本地 AI 聊天 | 對話式檢索整個知識庫 |
| **Transcribe** | 語音轉文字 | 本地 Whisper 模型，會議錄音秒轉 Markdown |
| **BRAT** | 提前測試新插件 | 不用等官方審核就能裝最新插件 |

## 六、安裝建議

| 優先級 | 建議 |
|--------|------|
| **第一步** | 先熟悉內建功能（反向連結、關係圖、快速切換），不要一上來就裝一堆插件 |
| **第二步** | 裝 Templater + Periodic Notes，建立每日筆記習慣 |
| **第三步** | 筆記累積到 50+ 篇後，再裝 Dataview 做自動化查詢 |
| **第四步** | 根據實際痛點挑選第二梯隊插件，不要一次性全裝 |

> **重點提醒：** 插件是工具，不是目的。Obsidian 真正的價值在於**雙向連結帶來的知識複利**——筆記越多，關聯越強，產生的洞察就越多。
