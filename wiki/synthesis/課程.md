# 課程

> **核心命題：** 解決老闆缺人才問題 → 建構各公司自己的 SOP + AI 大腦 → 人變成 AI 的手腳
> **包裝策略：** 對老員工說「省力」，對老闆說「留資產」

---

## 為什麼現在要開始用 AI+人協作去思考？

### 時代背景

1. **缺工不是暫時現象，是長期趨勢** — 少子化、高齡化，年輕人不想進傳統產業
2. **經驗帶不走** — 老員工退休，經驗就消失了
3. **AI 已經到了「能用」的門檻** — 不是科幻電影，是明天就能用的工具
4. **先用的公司會贏** — 不是技術領先，是「用 AI 把經驗留下來」的公司會活得好

### 五階段演進

| 階段 | 名稱 | 說明 |
|---|---|---|
| 1 | **AI 工具人** | 用 AI 寫文章、做圖、回訊息（你現在可能就在這） |
| 2 | **AI 知識庫** | 把 SOP、流程、經驗餵給 AI，讓它能問能答 |
| 3 | **AI 協作者** | AI 不只回答，還能主動建議、執行任務 |
| 4 | **AI 團隊** | 多個 AI agent 分工合作，像一支數位團隊 |
| 5 | **AI+人協作** | 人做決策、AI 做執行，互補長處 |

**這堂課的目標：帶你從階段 1 走到階段 4，建立你自己的 AI 團隊。**

### 六階段操作手冊（建立公司 AI 資產的 6 個步驟）

> **核心邏輯：Brain 先完成，Agent 才執行。Agent 不用自己猜，Brain 已經告訴他怎麼做。**

```
能力拆解 → Skill 建立 → Skill 驗證 → Project 組裝 → Company Brain → Agent 執行
```

| 步驟 | 名稱 | 說明 |
|---|---|---|
| 1 | **能力拆解**（Capability Extraction） | 找出公司真正有價值、值得被複製的核心能力（銷售、社群、招募、管理、SOP、教學…不是所有能力都拆，先找核心） |
| 2 | **Skill 建立**（Skill Packaging） | 把能力拆成一個一個 Skill 包（Threads Skill、FB Skill、成交 Skill、客服 Skill…） |
| 3 | **Skill 驗證**（Skill Validation） | 去市場驗證——真的有曝光、留言、成交，才正式納入。不是拆完就是 Skill |
| 4 | **Project 組裝**（Project Assembly） | 加入品牌、公司、商品、世界觀、SOP，把多個 Skill 組成完整專案（例如：AI 社群代運營 Project） |
| 5 | **Company Brain**（Company Brain） | 公司的知識、Skill、SOP、品牌、文化全部組合成公司的 AI 大腦。**Brain 應該先完成，Agent 才執行** |
| 6 | **Agent 執行**（Agent Execution） | Agent 開始讀 Company Brain，不用自己猜。例如：客服 Agent 讀取客服 Skill 來回答，社群 Agent 讀取 Threads Skill 來寫文 |

**五階段回答「在哪」，六階段回答「怎麼去」。**

## 課程教什麼

本堂課教的是「運作邏輯」，不是「工具教學」。

- ✅ 教的是：怎麼思考、怎麼萃取、怎麼建置
- ❌ 不教的是：用哪個工具、哪個模型、哪個平台

**因為工具會變，邏輯不會。** 學會邏輯，換任何工具都能用。

## 一、核心問題與解法

### 問題
- 老闆缺人才，但人才難培育、難複製
- 老員工經驗在腦袋裡，離職就消失
- 新人上手慢，依賴口傳心授
- SOP 寫了沒人看，看了也不會用

### 解法
```
老員工腦袋（隱性知識）
    ↓ ① 聊天式萃取（老員工零負擔）
結構化 SOP / 知識庫（顯性知識）
    ↓ ② 餵給 AI
AI 大腦（RAG + Agent）
    ↓ ③ AI 當導師
新人（AI 的手腳 + 監督者）
    ↓ ④ 反饋迴路
經驗累積 → SOP 更新 → AI 更準
```

## 二、老員工包裝策略

### 核心心法
- **老員工的動機：**「省力」
- **老闆的收獲：**「留知識」
- **兩邊都贏，但說法完全不同**

### 錯誤說法 vs 正確說法

| 錯誤說法（觸發防備） | 正確說法（觸發渴望） |
|---|---|
| 「把經驗整理成 SOP」 | 「幫你找一個 AI 助理，替你做事」 |
| 「把知識留下來」 | 「你只需要教我怎麼做，AI 幫你記」 |
| 「公司要建立知識庫」 | 「新人有事問 AI，不用打擾你」 |

### 心理學底層

| 老員工的恐懼 | 你的包裝 | 真實目的 |
|---|---|---|
| 我被取代 | AI 幫你省力 | 提取知識 |
| 白工沒有報酬 | AI 成為你的助理 | 建立 SOP |
| 新人學不會 | AI 帶新人，不用你教 | 知識傳承 |
| 我又在加班 | 一次設定，永久受益 | 低負擔參與 |

## 三、四階段操作流程

### 階段一：聊天式萃取（老員工以為只是在聊天）

**你的角色：** 像顧問一樣訪問，實際在「錄音 + 整理」

| 你問的 | 老員工以為的 | 實際在做的 |
|---|---|---|
| 「上次客戶臨時改規格，你怎麼處理？」 | 分享經驗 | 提取決策邏輯 → SOP 分支 |
| 「怎麼判斷案子該接還是不接？」 | 討論業務 | 提取評估標準 → AI 決策樹 |
| 「新人最常問你什麼？」 | 吐槽新人 | 提取 FAQ → AI 問答庫 |
| 「你每天第一件事做什麼？」 | 聊日常 | 提取工作流 → AI 任務清單 |

**關鍵：老員工不需要寫任何東西。**

### 階段二：AI 助理上線（老員工看到好處）

```
老員工的 AI 助理功能：
1. 提醒：「明天要寄報價單，模板在 XX」
2. 建議：「這個客戶類似上月 B 案，參考嗎？」
3. 草稿：「郵件草稿在這裡，你確認就好」
4. 查詢：「上次 C 客戶的合約條款？→ 自動回覆」
5. 備忘：「這個客戶不喜歡電話，只接受 LINE」
```

### 階段三：新人接手（老員工安心）

```
新人第一天：
AI：「你是新來的 XX，主管是 XX。
 第一週要做的事：
 1. 上午：讀這 3 份文件
 2. 下午：跟著主管跑拜訪
 3. 問題直接問我，我知道 80% 的答案」

新人問 AI：「報價怎麼算？」
AI：「步驟 ABC...
 上次主管加了 5% 緩衝，可參考。」
```

### 階段四：持續進化（知識螺旋）

```
新人執行 → AI 建議 → 結果記錄 → AI 學習 → SOP 更新
                                        ↑
                                    資深員工驗證
```

遵循 **SECI 模型**：
- **外化：** 隱性知識 → SOP（AI 整理）
- **組合：** 多個 SOP → AI 大腦（AI 串接）
- **內化：** 新人用 AI → 學到手感（只能靠人）
- **社會化：** 新人經驗 → 回饋 AI → 更新 SOP（閉環）

## 四、技術架構

### 三層分工模型

```
第 3 層：決策層（人）
  最終拍板、客戶關係、創意方向

第 2 層：判斷層（AI 建議 + 人確認）
  方案評估、風險判斷、資源分配

第 1 層：執行層（AI 執行 + 人監督）
  資料整理、文案撰寫、客服回覆、報表製作
```

### 技術選項（由淺入深）

| 級別 | 方案 | 成本 | 效果 |
|---|---|---|---|
| 🟢 MVP | ChatGPT Custom GPT + SOP 文件 | ~NT$22/月 | 能問答，小團隊 |
| 🟡 進階 | Dify / AnythingLLM + RAG | 自建，低成本 | 可串接內部系統 |
| 🔴 完整 | Hermes Agent + Skills 系統 | 你已有 | 真正的 AI Agent 團隊 |

### 你的殺手鐧
Hermes Agent 的 Skills 系統天然適合：
- 每個 SOP = 一個 Skill
- Skill = 角色定義 + 流程 + 知識庫
- 新人 = Agent，AI 大腦 = 系統提示詞 + Skills
- **不是「能問答的知識庫」，是「能執行任務的 AI 員工」**

## 課程定價策略（2026-06-30 更新）

### 業老闆的思考過程

- **實體銷售強，線上銷售未驗證** — 以前主要靠實體銷售，現在轉線上，不確定能否達成同樣效果
- **容易超時、為了負責任給很多** — 需要外部系統設定服務上限（人數/時間/課後回覆期限）
- **傾向低價（NT$1,000半天）** — 但擔心太低沒有行銷預算，無法跟其他管道合作或打廣告
- **低價的死循環風險：** 定價太低 → 沒有預算 → 招不到人 → 更便宜

### 蝦趴獅建議

- **種子場定價：NT$1,000** — 前兩期，但要學員簽同意書可取素材（見證+案例）
- **常規場定價：NT$2,980** — 第三起以後，正式定價
- **服務上限設計：**
  - 半天課只招 15 人（人數上限，外地可來）
  - 課後回覆只限 3 天（時間上限）
  - 不提供 1 對 1 諮詢（服務上限）
  - 寫在招生頁面：「本課程不提供課後一對一輔導」
- **課程產品矩陣：** 半天主課（NT$1,000-2,980）→ 應用小課（NT$200-300/堂，按場地費計）→ 企業內訓（NT$100K+）→ 顧問專案
- **行銷預算：** 定價 10-15% 撥行銷，用於測試管道有效性

### 定價框架

| 階段 | 定價 | 目的 |
|---|---|---|
| 第 1-2 期（種子班） | NT$1,000 | 累積案例、學員見證、名聲，條件：同意取素材 |
| 第 3 期以後 | NT$2,980 | 正式定價 |
| 早鳥價 | 標準價 × 0.8 | 製造 urgency |
| 團體價（3+） | 標準價 × 0.85 | 鼓勵企業帶人來 |
| 應用小課 | NT$200-300/堂 | 按場地費計，單堂報名 |
| 企業內訓 | NT$100,000+/場 | 高單價產品 |

### 應用小課設計

主課教「運作邏輯」，小課教「具體應用」。

**小課特色：**
- 每次只解決一個主題問題
- 單堂報名，場地費 NT$200-300
- 學員可以只來聽自己需要的，不用買整套
- 小課 = 從主課延伸出來的應用篇

**小課主題範例：**
- 小課 ①：用 AI 寫 FB 貼文（主課教提示詞，小課教實際寫）
- 小課 ②：用 AI 做社群數據分析（主課教心智模型，小課教具體工具）
- 小課 ③：用 AI 萃取你的 SOP（主課教萃取方法，小課帶學員做自己的案例）
- 小課 ④：用 AI 做客戶管理 CRM（主課教知識庫建置，小課教具體設定）

**商業模式：**
- 主課是入門（NT$1,000-2,980）
- 小課是利潤來源（NT$200-300/堂，但累積起來很多）
- 企業內訓是高單價（NT$100K+）
- 顧問專案是終極產品

## 實體課報名驅動因素（2026-07-07 新增）

### 核心洞察

**線上的人願意報名實體課，是因為實體課提供了線上學不到的獨家價值：**

1. **AI 模型配比 CP 值分析** — 實體課才會教：哪一個模型配比 CP 值最高，怎麼用最省錢又最有效
2. **免費 API 資源** — 實體課提供可以免費使用的 API 清單與設定方法
3. **即時互動與客製化** — 現場針對學員自己的公司做模型選擇與配置建議

### 商業意義

| 線上 | 實體 |
|------|------|
| 教「運作邏輯」 | 教「模型配比 + 免費 API」 |
| 通用方法論 | 獨家資源 + 現場診斷 |
| 可複製的 Skill 包 | 個人化 AI 團隊建置藍圖 |

**實體課是線上內容的「獨家升級包」，不是重複內容。** 這解釋了為什麼免費線上內容很多，但還是有人願意花錢來實體課——因為實體課有「獨家資源」作為鉤子。

### 應用方向

- 招生頁面明確列出實體課獨享內容（模型配比、免費 API）
- 線上內容可以吸引流量，實體內容負責變現
- 實體課學員後續可轉企業內訓（高單價產品）

### 市場現況總覽

台灣 AI 實體課程市場分成三個價格帶：

| 價格帶 | 範圍 | 典型內容 | 代表課程 |
|---|---|---|---|
| **低價引流** | NT$1,500-3,000 | 工具操作、ChatGPT 基礎 | ACCUPASS AI 全方位應用 NT$2,980（4 小時） |
| **中價實戰** | NT$4,000-7,000 | 任務導向、知識庫、專案實作 | 大人學 Esor NT$5,980（半天）、Abby 影音 NT$4,800（4 小時） |
| **高價企業級** | NT$10,000+ | 企業內訓、策略導入 | 工研院 NT$9,700（1 天）、戰國策企業內訓 |
