# AI 數位團隊建置 × 課程架構整合方案

> 整合日期：2026-06-19
> 核心命題：解決老闆缺人才問題 → 建構各公司自己的 SOP + AI 大腦 → 人變成 AI 的手腳
> 包裝策略：對老員工說「省力」，對老闆說「留資產」

---

## 一、核心問題與解法

### 問題
- 老闆缺人才，但人才難培育、難複製
- 老員工經驗在腦袋裡，離職就消失
- 新人上手慢，依賴口傳心授
- SOP 寫了沒人看，看了也不會用

### 解法
```
老員工腦袋（隱性知識）
    ↓ ① 聊天式萃取（老員工零負擔）
結構化 SOP / 知識庫（顯性知識）
    ↓ ② 餵給 AI
AI 大腦（RAG + Agent）
    ↓ ③ AI 當導師
新人（AI 的手腳 + 監督者）
    ↓ ④ 反饋迴路
經驗累積 → SOP 更新 → AI 更準
```

## 二、老員工包裝策略

### 核心心法
- **老員工的動機：**「省力」
- **老闆的收獲：**「留知識」
- **兩邊都贏，但說法完全不同**

### 錯誤說法 vs 正確說法

| 錯誤說法（觸發防備） | 正確說法（觸發渴望） |
|---|---|
| 「把經驗整理成 SOP」 | 「幫你找一個 AI 助理，替你做事」 |
| 「把知識留下來」 | 「你只需要教我怎麼做，AI 幫你記」 |
| 「公司要建立知識庫」 | 「新人有事問 AI，不用打擾你」 |

### 心理學底層

| 老員工的恐懼 | 你的包裝 | 真實目的 |
|---|---|---|
| 我被取代 | AI 幫你省力 | 提取知識 |
| 白工沒有報酬 | AI 成為你的助理 | 建立 SOP |
| 新人學不會 | AI 帶新人，不用你教 | 知識傳承 |
| 我又在加班 | 一次設定，永久受益 | 低負擔參與 |

## 三、四階段操作流程

### 階段一：聊天式萃取（老員工以為只是在聊天）

**你的角色：** 像顧問一樣訪問，實際在「錄音 + 整理」

| 你問的 | 老員工以為的 | 實際在做的 |
|---|---|---|
| 「上次客戶臨時改規格，你怎麼處理？」 | 分享經驗 | 提取決策邏輯 → SOP 分支 |
| 「怎麼判斷案子該接還是不接？」 | 討論業務 | 提取評估標準 → AI 決策樹 |
| 「新人最常問你什麼？」 | 吐槽新人 | 提取 FAQ → AI 問答庫 |
| 「你每天第一件事做什麼？」 | 聊日常 | 提取工作流 → AI 任務清單 |

**關鍵：老員工不需要寫任何東西。**

### 階段二：AI 助理上線（老員工看到好處）

```
老員工的 AI 助理功能：
1. 提醒：「明天要寄報價單，模板在 XX」
2. 建議：「這個客戶類似上月 B 案，參考嗎？」
3. 草稿：「郵件草稿在這裡，你確認就好」
4. 查詢：「上次 C 客戶的合約條款？→ 自動回覆」
5. 備忘：「這個客戶不喜歡電話，只接受 LINE」
```

### 階段三：新人接手（老員工安心）

```
新人第一天：
AI：「你是新來的 XX，主管是 XX。
 第一週要做的事：
 1. 上午：讀這 3 份文件
 2. 下午：跟著主管跑拜訪
 3. 問題直接問我，我知道 80% 的答案」

新人問 AI：「報價怎麼算？」
AI：「步驟 ABC...
 上次主管加了 5% 緩衝，可參考。」
```

### 階段四：持續進化（知識螺旋）

```
新人執行 → AI 建議 → 結果記錄 → AI 學習 → SOP 更新
                                        ↑
                                    資深員工驗證
```

遵循 **SECI 模型**：
- **外化：** 隱性知識 → SOP（AI 整理）
- **組合：** 多個 SOP → AI 大腦（AI 串接）
- **內化：** 新人用 AI → 學到手感（只能靠人）
- **社會化：** 新人經驗 → 回饋 AI → 更新 SOP（閉環）

## 四、技術架構

### 三層分工模型

```
第 3 層：決策層（人）
  最終拍板、客戶關係、創意方向

第 2 層：判斷層（AI 建議 + 人確認）
  方案評估、風險判斷、資源分配

第 1 層：執行層（AI 執行 + 人監督）
  資料整理、文案撰寫、客服回覆、報表製作
```

### 技術選項（由淺入深）

| 級別 | 方案 | 成本 | 效果 |
|---|---|---|---|
| 🟢 MVP | ChatGPT Custom GPT + SOP 文件 | ~NT$22/月 | 能問答，小團隊 |
| 🟡 進階 | Dify / AnythingLLM + RAG | 自建，低成本 | 可串接內部系統 |
| 🔴 完整 | Hermes Agent + Skills 系統 | 你已有 | 真正的 AI Agent 團隊 |

### 你的殺手鐧
Hermes Agent 的 Skills 系統天然適合：
- 每個 SOP = 一個 Skill
- Skill = 角色定義 + 流程 + 知識庫
- 新人 = Agent，AI 大腦 = 系統提示詞 + Skills
- **不是「能問答的知識庫」，是「能執行任務的 AI 員工」**

## 五、課程架構整合

### 課程定位
**「AI 數位團隊建置實戰」** — 教老闆/主管如何建構自己公司的 AI 團隊

### 目標學員
- 微小型公司老闆/高階主管
- 想要創業的人（一人公司）
- 需要帶團隊的主管

### 課程大綱（4 堂 × 2 小時）

---

#### 第 1 堂：AI 思維重塑 — 從工具到夥伴

**目標：** 打破對 AI 的錯誤想像，建立正確心智模型

| 時間 | 內容 | 形式 |
|---|---|---|
| 0-15 min | 開場：你公司最值錢的資產是什麼？（答案是：人的經驗） | 提問互動 |
| 15-30 min | AI 的三個盲區：幻覺、失憶、討好 | 講解 |
| 30-45 min | 瓦基 PACE 框架簡化版：AI 的四種角色 | 講解 + 範例 |
| 45-60 min | SCORE 提示詞框架：從 60 分到 90 分 | 講解 |
| 60-90 min | **實作：** 用 SCORE 寫出第一個提示詞 | 學員動手 |
| 90-105 min | 風格指南：讓 AI 活得像你，不是像你活得像 AI | 講解 |
| 105-120 min | **帶走成品：** 你的第一個 AI 提示詞模板 | 實作產出 |

**帶走成品：** 3 個可直接用的提示詞模板（用於日常工作中）

---

#### 第 2 堂：SOP 萃取實戰 — 把經驗變成可複製的流程

**目標：** 學會用「聊天式萃取法」把老員工經驗轉成 SOP

| 時間 | 內容 | 形式 |
|---|---|---|
| 0-15 min | 複習上堂 + 學員分享提示詞使用經驗 | 分享 |
| 15-35 min | 知識萃取四步驟：訪談 → 案例 → SOP → 驗證 | 講解 |
| 35-55 min | 聊天式萃取法：怎麼問、怎麼追、怎麼整理 | 示範 |
| 55-75 min | **實作：** 學員拿出自己公司的真實案例，现场萃取 | 小組練習 |
| 75-95 min | AI 輔助整理：把訪談記錄變成結構化 SOP | 示範 + 實作 |
| 95-110 min | 高頻/高影響力/難複製：優先萃取什麼？ | 講解 |
| 110-120 min | **帶走成品：** 你自己的 SOP 初稿 | 實作產出 |

**帶走成品：** 1-3 份自己公司的 SOP 初稿 + AI 整理模板

---

#### 第 3 堂：AI 大腦建置 — 從 SOP 到能問能做的系統

**目標：** 把 SOP 裝進 AI 系統，讓它變成公司的大腦

| 時間 | 內容 | 形式 |
|---|---|---|
| 0-15 min | 複習上堂 SOP + 學員分享 | 分享 |
| 15-30 min | RAG 原理白話版：怎麼讓 AI 讀你的文件 | 講解 |
| 30-50 min | 三種建置方案比較（MVP/進階/完整） | 講解 |
| 50-70 min | **實作：** 用 ChatGPT Custom GPT 建立第一個知識庫 | 學員動手 |
| 70-90 min | 三層分工模型：什麼讓 AI 做、什麼讓人做 | 講解 + 討論 |
| 90-105 min | 反饋機制：怎麼讓 AI 越用越準 | 講解 |
| 105-120 min | **帶走成品：** 你的第一個 AI 知識庫（Custom GPT） | 實作產出 |

**帶走成品：** 一個可運作的 AI 知識庫（Custom GPT 或 Dify）

---

#### 第 4 堂：新人帶教與持續進化 — 讓 AI 團隊自己成長

**目標：** 建立新人 AI 帶教系統 + 知識螺旋機制

| 時間 | 內容 | 形式 |
|---|---|---|
| 0-15 min | 複習上堂知識庫 + 學員分享使用經驗 | 分享 |
| 15-35 min | 新人第一天 AI 帶教流程設計 | 講解 + 範例 |
| 35-55 min | **實作：** 設計你自己公司的「新人 AI 帶教清單」 | 學員動手 |
| 55-75 min | 知識螺旋（SECI）：怎麼讓系統持續進化 | 講解 |
| 75-95 min | 每月維運機制：檢視、更新、優化 | 講解 |
| 95-110 min | 綜合實作：把你的 SOP + 知識庫 + 帶教清單串起來 | 實作 |
| 110-120 min | 總結 + 下一步行動計畫 | 分享 |

**帶走成品：** 完整的 AI 團隊建置藍圖（SOP + 知識庫 + 帶教系統 + 維運計畫）

---

### 課程特色

1. **每堂都有帶走成品** — 不是聽完就忘，是帶著東西走
2. **用自己的公司當教材** — 學完直接能用
3. **不教工具，教方法** — 工具會換，方法論不會
4. **老員工包裝策略** — 這是你的獨家差異化內容

## 課程定價策略（2026-06-30 更新）

### 業老闆的思考過程

- **實體銷售強，線上銷售未驗證** — 以前主要靠實體銷售，現在轉線上，不確定能否達成同樣效果
- **容易超時、為了負責任給很多** — 需要外部系統設定服務上限（人數/時間/課後回覆期限）
- **傾向低價（NT$1,000半天）** — 但擔心太低沒有行銷預算，無法跟其他管道合作或打廣告
- **低價的死循環風險：** 定價太低 → 沒有預算 → 招不到人 → 更便宜

### 蝦趴獅建議

- **種子場定價：NT$2,980** — 低於 3,000 心理門檻，但不至於賤賣
- **服務上限設計：**
  - 半天課只招 12 人（人數上限）
  - 課後回覆只限 3 天（時間上限）
  - 不提供 1 對 1 諮詢（服務上限）
  - 寫在招生頁面：「本課程不提供課後一對一輔導」
- **課程產品矩陣：** 半天主課 → 小課深化 → 企業內訓 → 顧問專案
- **行銷預算：** 定價 10-15% 撥行銷，用於測試管道有效性

### 定價框架

| 階段 | 定價 | 目的 |
|---|---|---|
| 第 1 堂（種子場） | NT$2,980 | 累積案例、學員回饋、名聲 |
| 第 2-3 堂 | NT$3,980 | 驗證市場接受度 |
| 第 4 堂以後 | NT$5,980 | 正常定價，開始投廣告 |
| 早鳥價 | 標準價 × 0.8 | 製造 urgency |
| 團體價（3+） | 標準價 × 0.85 | 鼓勵企業帶人來 |
