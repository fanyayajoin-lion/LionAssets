# OpenClaw vs Hermes Agent 競品分析

## 核心結論

**快速落地選 OpenClaw，長期陪伴選 Hermes。**

## 對比表

| 維度 | OpenClaw | Hermes Agent |
|------|----------|-------------|
| 設計哲學 | Gateway 架構，連接與協調 | 閉環學習循環，自我進化 |
| 記憶 | 全量存，什麼都留 | 有限記憶，壓縮提純 |
| 技能 | 他人編寫，公共教材 | Agent 自己生成維護，私人筆記 |
| 上手難度 | 低，配置化 | 中等偏陡 |
| 中文生態 | 極強（微信/QQ/釘釘原生） | 一般 |
| 模型切換 | 推薦自家模型 | 徹底解耦，18+ 供應商 |

## 市場定位洞察

### OpenClaw 優勢
- 中文生態完整（微信、QQ、釘釘原生支援）
- 配置化上手快，適合不想碰技術的老闆
- Gateway 架構強調「連接」而非「學習」
- 公共 Skill 生態成熟

### Hermes 優勢
- 自我進化能力（學習循環、記憶壓縮提純）
- 模型供應商完全解耦（不綁定單一 LLM）
- Agent 自主生成 Skill，非依賴他人編寫
- 適合長期深度使用、個人化程度高的場景

### 對群獅的意義
1. **競品定位確認**：OpenClaw 是「快速落地型」，Hermes 是「長期陪伴型」。這跟群獅的顧問定位一致——我們賣的不是工具，是可持續進化的系統。
2. **中文生態是弱項**：如果老闆要推 Hermes 相關課程或顧問服務，需要補強中文社群內容。
3. **36氪報導是正面信號**：Hermes Star 數超 35k，說明市場認可度在上升。
