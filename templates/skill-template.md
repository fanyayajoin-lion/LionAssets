# 品牌案例拆解 Skill 模板

## Purpose
拆解品牌案例，萃取可複製的商業框架。

## When to use
- 收到新品牌案例時
- 需要分析競爭對手時
- 準備課程內容時

## When NOT to use
- 品牌資料不完整時（先補充資料）
- 客戶明確要求不做案例分析時

## Input
- 品牌名稱
- 產業別
- 案例資料（PPT、文章、截圖）

## Output
- 商業模式一句話
- 現有優勢
- 主要卡點
- AI 可放大的地方
- 可做成顧問服務的機會

## Conditions
- 品牌必須有明確的產業分類
- 案例資料至少包含：商业模式、客戶來源、主要產品

## Execution Steps
1. 確認品牌資料完整性
2. 套用五問框架拆解
3. 輸出標準格式
4. 回流 wiki/cases/
5. 更新 index.md

## Quality Standard
- 每個案例必須有 ≥2 個雙向連結
- 輸出必須包含「AI 可放大的地方」
- 避免只做流水帳摘要

## Example
見：skills/傳產案例拆解.md

## Common Mistakes
- 只整理資料，不做判斷
- 忘記連結到既有知識庫
- 輸出格式不一致
