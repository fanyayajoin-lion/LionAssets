# 品牌案例補全歸檔 Workflow

## 目的

任何 PPT / PDF 丟進 `assets/brand-cases/` 後，AI 不能只整理原檔內容，必須先補齊品牌最新資料、商業模式、近況與可延伸應用，再歸檔成可用資產。

## 觸發條件

- 使用者提供品牌案例 PPT/PDF。
- `assets/brand-cases/品牌名/` 出現新檔。
- 任何資料明顯是品牌、企業、商業模式案例。

## 步驟

1. 預設不複製原始檔；保留原始路徑，將摘要與補全結果寫進知識庫。
2. 只有老闆明確要求保存原檔時，才複製到 `assets/brand-cases/品牌名/`，且不覆蓋、不刪除。
3. 讀取原檔，產出原始重點摘要、原 PPT 使用框架、已過期或待查證資訊。
4. 連網補充最新資料：
   - 品牌現況
   - 近年營收 / 融資 / IPO / 門市 / 會員 / 通路
   - 最新產品線
   - 最新流量打法
   - 私域、自媒體、直播、電商策略
   - AI / 數位化應用
   - 可查證來源連結
5. 保留「原 PPT 觀點」與「最新補充」兩層，不用新資料覆蓋舊觀點。
6. 產出 `wiki/cases/產業分類/品牌名.md`。
7. 產出或更新 `wiki/entities/品牌名.md`。
8. 蒸餾到 `wiki/synthesis/`：
   - 可複製打法
   - 可延伸商業模式
   - 對傳產老闆的啟發
   - 對一人公司 / OPC 的啟發
   - 可變成文章、課程、顧問工具的素材
9. 更新 `index.md`、`hot.md`、`log.md`。

## 禁止

- 禁止只摘要 PPT 就歸檔。
- 禁止沒有來源就硬補最新資料。
- 禁止把過期資訊當作現況。
