# 安全與隱私設定

## 一句話理解
Hermes 會記住你的所有事情，所以要**保護好自己的資料**。

## 這是什麼
安全設定有三件事：
1. **保護 API Key** — 不要讓別人看到
2. **保護記憶** — 不要讓別人知道你的秘密
3. **保護備份** — 資料不要丟失

## 為什麼需要
Hermes 會記住你的：
- 個人資料（名字、職業、喜好）
- 客戶資料（名字、需求、交易記錄）
- 工作資料（報告、流程、方法論）

這些都是**商業機密**，不能讓別人看到。

## 生活比喻
想像你的**保險箱**：
- API Key = 保險箱鑰匙
- 記憶 = 保險箱裡的文件
- 備份 = 保險箱的影本

你要做的是：
1. 把鑰匙藏好
2. 把文件鎖好
3. 把影本放在安全的地方

## 操作步驟

### 步驟 1：保護 API Key
API Key 存在 `~/.hermes/.env` 檔案裡。
**不要分享這個檔案給任何人**。

### 步驟 2：設定備份
定期備份 `~/.hermes/` 目錄到雲端硬碟或 GitHub 私人 repo。

### 步驟 3：檢查隱私設定
輸入：
```bash
hermes config set privacy.redact_pii true
```
這樣 Hermes 會自動隱藏你的個人資訊。

## 常見錯誤

### 錯誤 1：把 API Key 貼在公開群組
千萬不要。如果有人看到你的 Key，他就可以用你的帳號。

### 錯誤 2：不備份
如果電腦壞了，記憶和技能都會消失。

### 錯誤 3：讓別人用你的 Hermes
Hermes 會記住你的所有事情。不要讓別人用你的帳號。

## 壞掉會怎樣
- **API Key 外洩**：馬上到 AI 模型官網重新產生 Key
- **記憶外洩**：清除記憶，重新設定
- **備份遺失**：從雲端或 GitHub 恢復

## 技術補充
- API Key 路徑：`~/.hermes/.env`
- 設定備份：`cp -r ~/.hermes ~/.hermes-backup`
- 檢查隱私：`hermes config set privacy.redact_pii true`
- 清除記憶：`hermes memory clear`
