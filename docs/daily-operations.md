# 日常維護、升級、備份

## 一句話理解
Hermes 不需要每天照顧，但**一週做一次檢查**就能讓他一直好好工作。

## 這是什麼
日常維護有三件事：
1. **升級** — 讓 Hermes 用最新版本
2. **備份** — 防止資料遺失
3. **檢查** — 確認 Hermes 還在好好工作

## 為什麼需要
Hermes 會不斷更新，新版本更好用、更安全。
但如果你不升級，可能會錯過重要功能。

## 生活比喻
想像你的**汽車**：

| 維護項目 | 汽車比喻 | Hermes 指令 |
|---------|---------|------------|
| 升級 | 定期保養 | `hermes update` |
| 備份 | 買保險 | 備份 `~/.hermes/` 目錄 |
| 檢查 | 看儀表板 | `hermes doctor` |

一週檢查一次，車子就不會半路拋錨。

## 操作步驟

### 步驟 1：檢查狀態（一週一次）

輸入：
```bash
hermes doctor
```

如果看到全是 ✅，表示一切正常。
如果有 ⚠️ 或 ❌，看提示修復就好。

### 步驟 2：升級（有新版本時）

輸入：
```bash
hermes update
```

升級過程大約 2-5 分鐘。升級後重新開啟 Hermes 就會生效。

### 步驟 3：備份（一週一次）

Hermes 的所有資料都存在 `~/.hermes/` 目錄。
只要把這個目錄複製到其他地方，就是備份。

**推薦做法**：
- 備份到雲端硬碟（Google Drive、Dropbox）
- 或備份到 GitHub（私人 repo）

## 常見錯誤

### 錯誤 1：太久沒升級
Hermes 更新很快，建議**有通知就升級**。
舊版本可能會錯過新功能或安全修復。

### 錯誤 2：不備份
如果電腦壞了，你的記憶和技能都會消失。
**至少每月備份一次**。

### 錯誤 3：升級後發現不好用
新版本偶爾有小問題。如果遇到：
```bash
hermes update --rollback
```
可以回退到上一個版本。

## 壞掉會怎樣
- **不升級**：Hermes 還是能用，只是沒有新功能
- **不備份**：資料會丟失，但 Hermes 本身沒事
- **升級失敗**：重試就好，通常不會有問題

## 技術補充
- 檢查路徑：`hermes doctor`
- 升級指令：`hermes update`
- 備份路徑：`~/.hermes/`
- 日誌位置：`~/.hermes/logs/`
- 診斷工具：`hermes dump`（貼給技術支援用）
