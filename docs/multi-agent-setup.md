# 多 Agent 架構

## 一句話理解
你可以請**一群 AI 助理**一起工作，每個人負責不同的事。

## 這是什麼
**多 Agent** 就是讓多個 Hermes 同時工作。

想像你有一間**小公司**：
- **小美**：負責客戶服務
- **阿強**：負責專案管理
- **小黑**：負責資料分析

他們各司其職，但都聽你的指揮。

## 為什麼需要
- **分工合作**：每個 Agent 專精不同的領域
- **同時處理多件事**：小美回覆客戶時，阿強在做報告
- **不怕一個人累死**：多個人一起扛

## 生活比喻
就像**餐廳後廚**：

| Agent | 角色 | 工作 |
|-------|------|------|
| **小美** | 前廳主管 | 接待客戶、安排座位 |
| **阿強** | 主廚 | 做菜、控品質 |
| **小黑** | 倉管 | 管食材、盤點庫存 |

你（老闆）只需要告訴小美：「今天有 10 桌」，她會安排其他人去做。

## 實際使用場景

### 場景 1：客戶服務 Agent
專門回覆客戶訊息，語氣溫柔專業。

### 場景 2：專案管理 Agent
專門追蹤進度、提醒截止日期。

### 場景 3：資料分析 Agent
專門整理數據、做圖表、寫報告。

## 常見錯誤

### 錯誤 1：Agent 太多記不住
建議不要超過 5 個。太多了反而混亂。

### 錯誤 2：Agent 之間會打架
每個 Agent 有獨立的工作範圍，不會互相干擾。

## 壞掉會怎樣
- **一個 Agent 壞了**：其他 Agent 還在，不會全斷
- **全部壞了**：重啟就好

## 技術補充
- 建立 Agent：`hermes profile create <名稱>`
- 切換 Agent：`hermes profile use <名稱>`
- 查看所有 Agent：`hermes profile list`
