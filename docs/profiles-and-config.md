# 多個分身與設定檔

## 一句話理解
你可以有**多個 Hermes**，每個都有不同的個性、記憶和工作方法。

## 這是什麼
**Profile（分身）** 就是 Hermes 的不同版本。

想像你有三位秘書：
- **小美**：擅長回覆客戶，語氣溫柔
- **阿強**：擅長寫報告，風格嚴謹
- **小黑**：擅長分析資料，直接了當

他們都叫 Hermes，但各有各的專長。在 Hermes 的世界裡，每個分身就叫一個 **Profile**。

## 為什麼需要
- **工作/私人分開**：工作分身不記私人事，私人分身不記工作事
- **不同客戶用不同風格**：香料大叔的分身回覆客戶，命理 AI 的分身解盤
- **測試不怕搞壞**：先在測試分身練習，確認OK再套用到正式分身

## 生活比喻
想像你有一間**顧問公司**：

| Profile | 角色 | 專長 |
|---------|------|------|
| **小美** | 客戶服務 | 回覆 Email、安排會議 |
| **阿強** | 專案管理 | 追蹤進度、寫報告 |
| **小黑** | 資料分析 | 整理數據、做圖表 |

他們共用同一個辦公室（你的電腦），但各有各的工作範圍。

你也可以請**外包**：
- **小綠**（測試分身）：新 Skill 先在這裡試，OK 了再搬到正式分身

## 實際使用場景

### 場景 1：建立新分身

輸入：
```bash
hermes profile create 客戶服務
```

然後設定這個分身的個性：
```
你是客戶服務專員，語氣溫柔專業。
你擅長回覆客戶諮詢，不推銷，先了解需求。
```

### 場景 2：切換分身

輸入：
```bash
hermes profile use 客戶服務
```

以後你跟他說話，他就是客戶服務專員。

### 場景 3：查看所有分身

輸入：
```bash
hermes profile list
```

會列出所有分身和他們的狀態。

### 場景 4：刪除分身

輸入：
```bash
hermes profile delete 測試分身
```

## 常見錯誤

### 錯誤 1：分身太多記不住
建議不要超過 5 個分身。太多反而混亂。

### 錯誤 2：分身之間會互相干擾
不會。每個分身有獨立的記憶和技能。
但**設定檔是共用的**（比如 AI 模型），如果要分開設定，需要手動調整。

### 錯誤 3：以為分身就是不同的 AI
分身還是用同一個 AI 模型（比如 GPT-4o），只是**個性、記憶、技能不同**。

## 壞掉會怎樣
- **分身壞了**：刪除重建就好，不會影響其他分身
- **設定檔壞了**：Hermes 還在，只是個性变了。重新設定就好。
- **所有分身都用同一個 AI 模型**，所以模型壞了，所有分身都壞。

## 技術補充
- 分身路徑：`~/.hermes/profiles/<分身名稱>/`
- 每個分身有自己的：
  - `SOUL.md`（個性設定）
  - `memories/`（記憶）
  - `skills/`（技能）
- 共用設定：`~/.hermes/config.yaml`（AI 模型等）
