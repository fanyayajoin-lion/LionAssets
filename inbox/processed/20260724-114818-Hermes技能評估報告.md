# Hermes Agent 第三方技能評估報告

## 評估背景

老闆收到六個技能名稱，要求找出並分析是否需要安裝。

搜尋方法論：
1. 先用 `web_search` 搜尋各技能名稱 + "hermes skill" 關鍵字
2. 用 `web_extract` 抓取 GitHub repo、skills.sh、llmbase.ai 等來源頁面
3. 找不到就換關鍵字二次搜尋，直到抓到足夠資訊

## 評估結果總覽

| # | 技能名稱 | 來源 | 功能 | 建議 |
|---|---|---|---|---|
| 1 | markdown-converter | intellectronica/agent-skills (skills.sh) | 用 `uvx markitdown` 將 PDF/DOCX 等轉成 Markdown，零安裝 | ⚠️ 低優先級 |
| 2 | free-ride | dazeb/freeride-h (GitHub) | 跨 9 供應商、43+ 免費模型的自動切換管理器 | ❌ 不建議 |
| 3 | agent-browser | vercel-labs/agent-browser (Vercel Labs 官方) | Rust 寫的瀏覽器自動化 CLI，比內建 browser 更快更強 | ⚠️ 看需求 |
| 4 | desktop-control | agent-sh/computer-use-linux (GitHub) | Linux 桌面控制 MCP server，AT-SPI + 截圖 + 點擊 | ❌ 不適合 |
| 5 | self-improving | pskoett/self-improving-agent (skills.sh) | 記錄學習/錯誤到 .learnings/ 目錄，跨 session 改進 | ❌ 功能重複 |
| 6 | skill-creator | jixoai/skill-creator (GitHub) | 從 npm package 自動建立 Claude Code skills | ❌ 不適合 |

## 詳細分析

### 1. markdown-converter

**來源：** [intellectronica/agent-skills](https://github.com/intellectronica/agent-skills)（透過 skills.sh 分發）

**功能：** 使用 `uvx markitdown` 將 PDF、DOCX、PPTX 等檔案轉換為 Markdown。不需要額外安裝，直接用 uvx 執行。

**支援格式：** PDF、DOCX、PPTX、XLSX、HTML、EPUB、ZIP（含內部文件）

**安裝方式：**
```bash
npx skills add https://github.com/intellectronica/agent-skills --skill markdown-converter
```

**優點：** 輕量、零依賴、一行命令搞定

**缺點：** 知識庫已有 `ocr-and-documents` 技能（pymupdf/marker-pdf），功能更全面（含 OCR、表格解析、圖片提取）

**結論：** 如果你只需要簡單的 PDF → MD 轉換，這個夠用且輕量。但你已有更好的替代方案，優先級低。

---

### 2. free-ride (FreeRide-H)

**來源：** [dazeb/freeride-h](https://github.com/dazeb/freeride-h)（MIT License, Python 100%）

**功能：** 多供應商免費模型管理器。可瀏覽、切換、自動輪換 9 個供應商的 43+ 免費 AI 模型，API 耗完自動切下一個。

**支援供應商：** OpenRouter、OpenCode Zen、Mistral、Groq、Cerebras、SambaNova、NVIDIA NIM、Ollama Local、Ollama Cloud

**核心功能：**
- `freeride list` — 查看所有免費模型
- `freeride switch <model>` — 切換到指定模型
- `freeride-watcher` — 背景 daemon 自動輪換（每 60 秒 ping 一次，失敗或太慢就切）
- `freeride auto` — 一次性設定所有免費供應商

**優點：** 省 API 費用、自動故障轉移

**缺點：**
- 跟知識庫管理無關
- 需要額外配置 9 個供應商的 API key
- 模型品質不可控，自動切換可能影響任務品質
- 免費模型通常較舊或能力較弱

**結論：** 除非刻意想省錢且不在乎模型品質波動，否則不要裝。

---

### 3. agent-browser

**來源：** [vercel-labs/agent-browser](https://github.com/vercel-labs/agent-browser)（Vercel Labs 官方，38.9k stars）

**功能：** 快速瀏覽器自動化 CLI，專為 AI agent 設計。Rust 寫成，非 Node.js wrapper。

**核心優勢：**
- 比 Hermes 內建 browser 工具更快、更穩定
- Chrome/Chromium via CDP，無需 Playwright/Puppeteer
- Accessibility-tree snapshots + compact @eN element refs
- Sessions、authentication vault、state persistence、video recording
- 專用 skills：Electron app 控制（VS Code/Slack/Discord/Figma）、Slack 自動化、探索性 QA、Vercel Sandbox 雲端瀏覽器、AWS Bedrock AgentCore
- 可視化儀表板（port 4848）

**安裝方式：**
```bash
npm i -g agent-browser && agent-browser install
```

**適合場景：**
- 大量瀏覽器自動化（爬蟲、填表、批量操作）
- Electron 桌面 app 測試與自動化
- 探索性 QA / Bug hunt
- 在 Vercel Sandbox microVMs 中執行瀏覽器自動化

**結論：** 如果大量使用瀏覽器自動化，值得裝。如果只是偶爾瀏覽網頁，Hermes 內建 browser 已夠用。

---

### 4. desktop-control (computer-use-linux)

**來源：** [agent-sh/computer-use-linux](https://github.com/agent-sh/computer-use-linux)（MIT License, Rust 89.8%）

**功能：** Linux 桌面控制 MCP server。讓 AI agent 可以讀取 AT-SPI 無障礙樹、截圖、點擊、滾動、輸入文字。

**支援桌面環境：** GNOME Wayland/X11、KDE Plasma/KWin、Hyprland、i3、COSMIC

**核心功能：**
- AT-SPI 無障礙樹讀取（role, name, text, states, bounds）
- 螢幕截圖（Portal / GNOME Shell / KWin DBus）
- 視窗管理（activate, move, resize, focus）
- 輸入模擬（click, drag, press_key, type_text, perform_action）
- 終端機強化（交叉引用 TTY 與前景進程）

**安裝方式（Hermes）：**
```bash
hermes skills tap add agent-sh/computer-use-linux
hermes skills install agent-sh/computer-use-linux/computer-use-linux
```

**缺點：**
- 需要 X11/Wayland 桌面環境
- 你的環境是 headless 伺服器，沒有桌面
- 跟知識庫管理工作完全無關

**結論：** 完全不適合你的場景。

---

### 5. self-improving (self-improvement)

**來源：** [pskoett/self-improving-agent](https://github.com/pskoett/self-improving-agent)（skills.sh, 699 stars）

**功能：** 記錄學習、錯誤、使用者修正到 `.learnings/` 目錄的 markdown 文件，讓 agent 跨 session 持續改進。

**輸出文件：**
- `.learnings/LEARNINGS.md` — 修正、洞察、知識缺口、最佳實踐
- `.learnings/ERRORS.md` — 命令失敗與整合錯誤
- `.learnings/FEATURE_REQUESTS.md` — 使用者請求的功能

**優點：** 結構化記錄、可提升為 project memory、有 quality gates

**缺點：**
- 這只是「記錄」功能，不是真正的自我改進
- Hermes 已有內建 `memory` 工具和 `skill_manage` 做自我改進
- 這個技能專為 Claude Code / OpenClaw 設計，寫的是 `CLAUDE.md`、`SOUL.md`
- 跟 Hermes 的 curator 系統功能重疊

**結論：** 功能重複且針對其他工具，不要裝。

---

### 6. skill-creator

**來源：** [jixoai/skill-creator](https://github.com/jixoai/skill-creator)（MIT License）

**功能：** 自動從 npm package 建立 Claude Code skills 的 CLI + subagent。搜尋 package → 抓文件 → 建 skill 目錄。

**核心功能：**
- `search <keywords>` — 搜尋 npm packages
- `get-info <package>` — 取得套件詳細資訊
- `create-cc-skill <name>` — 建立新 skill 目錄
- `download-context7` — 下載並切片 Context7 文件
- `build-index` — 建立本地搜尋索引

**安裝方式：**
```bash
npm install -g skill-creator
```

**缺點：**
- 專為 **Claude Code** 設計，不是 Hermes Agent
- Hermes 已有內建 `skill_manage` 工具（create/patch/edit/delete/write_file/remove_file）
- 需要額外安裝 Context7 MCP server
- 依賴 npm/Node.js 生態系

**結論：** 功能重複且針對其他工具，不要裝。

## 最終建議

### 可考慮安裝（按優先級排序）

1. **agent-browser** — 唯一有明確實用價值的技能。如果你需要大量瀏覽器自動化、QA 測試、Electron app 控制，值得投資設定。
2. **markdown-converter** — 輕量但功能被現有 `ocr-and-documents` 覆蓋，除非你想要極簡方案。

### 不建議安裝

- **free-ride** — 省 API 錢的工具，與知識庫管理無關
- **desktop-control** — 需要桌面環境，你的伺服器是 headless
- **self-improving** — Hermes 內建 memory + skill_manage 已涵蓋
- **skill-creator** — 專為 Claude Code 設計，Hermes 有內建 skill_manage

## 搜尋方法論記錄

本次評估使用的搜尋方法：

1. **第一輪搜尋：** 用 `web_search` 搜尋每個技能名稱 + "hermes skill" 關鍵字
2. **內容擷取：** 用 `web_extract` 抓取 GitHub repo 頁面、skills.sh、llmbase.ai 的內容
3. **二次搜尋：** 找不到就換關鍵字再搜（如改搜 skills.sh、特定作者 repo）
4. **交叉驗證：** 對同一技能在不同來源交叉比對功能描述、安裝方式、star 數
